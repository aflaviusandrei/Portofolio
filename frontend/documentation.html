Welcome

Portofolio is a lightweight, responsive, highly-customizable content display framework and tool that makes it easier for you to get your content out there in a way that's accessible and beautiful.

Prerequisites

In order for Portofolio to work on your project, you'll need to include the following two CSS and JS files in your project:

portofolio.css

save_alt

init.js

save_alt 
<link rel="stylesheet" href="portofolio/portofolio.js">
<script src="portofolio/init.js">

Set up

There are two main ways of creating an album and displaying it on your project. While both involve writing code, both are very straightforward and easy to follow.

Create album through the form

You can use the intuitive album creation form on our front page. After naming your album, customizing it and linking to your materials in the order that you want (pictures, gifs, videos, audio files) you have two options:

  • Save the album to our database

    • You can choose to save the album to our database and keep it for later use. The album configuration (transition, navigation, fullscreen settings, etc.) will be saved, as well as its name and the media materials in the appropriate order.

    You'll be given a token that will allow you to call the album on any page you want via JavaScript with little hassle.


  • Generate the JSON and include it in your project as you see fit

    • You can choose to be given a ready-to-go JSON code that you can use to create a carousel on your own page.

Customize your JSON from scratch

You can use the guides and options in the "Display types" section to customize your album the way you want it, include the mark-up and initialize the function in your project yourself.

FAQ

Carousel

Mark-up

<div id="album">
   <div>
       <div><img src="1.jpg"/></div>
       <div><img src="2.jpg"/></div>
       <div><img src="3.jpg"/></div>
   </div>
</div>

Initializing

portofolio("album", {
   type : "carousel"
});

Configuration options

The carousel can be configured and customized in a number of different ways, all of which have defaults and are optional.

General settings

items

Type:

Number

Default:

3

The number of items per each slide.

speech

Type:

Bool

Default:

false

Turns speech commands off or on (currently only works on Chrome and requires microphone permissions).

Transition settings

transitionType

Type:

string

Default:

"slide"

Choose whether you want the entire slide to shift ("slide") or only a specific amount of items ("item").

transitionItems

Type:

Number

Default:

1

The amount of items to me moved, in case you choose the "item" transition type.

auto

Type:

Bool

Default:

false

Choose whether you want the carousel to move automatically in intervals.

autoInterval

Type:

Number

Default:

1500

The interval in milliseconds at which the carousel should move.

gestures

Type:

Bool

Default:

true

Turns swipe (mobile) and drag (desktop) off or on.

loop

Type:

Bool

Default:

true

Turns loop mode on or off.

Navigation settings

show

Type:

Bool

Default:

false

Turns navigation buttons on or off.

prev

Type:

String

Default: 

"<i>‹</i>"

String with a DOM element that you want to use as the "previous" button.

next

Type:

String

Default: 

"<i>›</i>"

String with a DOM element that you want to use as the "next" button.

Fullscreen settings

show

Type:

Bool

Default:

false

Turns fullscreen functionality on or off.

open

Type:

String

Default: 

"<i>⛶</i>"

String with a DOM element that you want to use as the "fullscreen" button.

close

Type:

String

Default: 

"<i>✖</i>"

String with a DOM element that you want to use as the "close" button.

Language settings (lang:)

next

Type:

String

Default:

"next"

The speech command that you want to use to move the carousel forward.

previous

Type:

String

Default:

"back"

The speech command that you want to use to move the carousel backwards.

code

Type:

String

Default:

"en-US"

The language code for the language you want to use. Here is a list of all available codes.

Responsive settings (responsive:)

breakpoint

Type:

Number

Default:

None

Specify a breakpoint of the screen width in pixels under which you want to use different settings for the carousel. You can change any of the aforementioned parameters under the new breakpoint.

Example:

responsive : {
   1200 : { // The following settings will be applied under 1200px
       items : 3,
       nav : {
           show : false
       },
       fullscreen : {
           show : false
       }
   }
}

Example of full carousel initialization using almost all parameters

<div id="album">
   <div>
       <div><img src="1.jpg"/></div>
       <div><img src="2.jpg"/></div>
       <div><img src="3.jpg"/></div>
   </div>
</div>


<script>
init("album", {
   type : "carousel",
   items : 4
   transition : {
       transitionType : "slide",
       transitionItems : 3,
       auto : true
       autoInterval : 1500
   },
   nav : {
       show : true,
       prev : "chevron_left",
       next : "chevron_right"
   },
   fullscreen : {
       show : true,
       open : "fullscreen",
       close : "close"
   },
   lang : {
       next : "следующий",
       prev : "предыдущий",
       lang : "ru"
   },
   responsive : {
       2000 : {
           transition : {
               auto : false
           },
           nav : {
               show : false
           }
       },
       800 : {
           items : 2,
           transition : {
               transitionType : "item",
               transitionItems : 2
           },
           fullscreen : {
               show : false
           }
       }
   }
});
</script>

Configuration options

The carousel can be configured and customized in a number of different ways, all of which have defaults and are optional.

General settings

itemsPerRow

Type:

Number

Default:

4

Number of items on each grid row.

fullscreen

Type:

Bool

Default:

false

Turn fullscreen mode on or off. Fullscreen mode allows your materials to be viewed in a gallery format similar to the fullscreen carousel.

format

Type:

String

Default:

None

This should be included and set to "video" if your gallery is composed of video elements in order to ensure player functionality.

Audio player

Mark-up

<div id="album">

</div>

There is no need for special mark-up, save for an empty div with an ID.

Initializing

portofolio("album", {
   type : "audio",
   songs : ["song1.mp3", "song2.mp3", "song3.mp3"]
});

The only other special parameter that the audio player needs is the list of songs in the form of an array of links.

Resize (grid extension)

As an extension, resize is optional and is not natively supported in the Portofolio core files, which is why you'll need to download an extra file.

Resize is an extension for the grid album option that allows the user to adjust the width of each grid element by dragging its margins as much as they want.

Prerequisites

There are two steps towards adding Resize to your album and enabling it.

1. Firstly, download the lightweight file for the extension and link to it inside your code:

resize.js

save_alt 
<script src="portofolio/resize.js">

2. Now, when you call the portofolio function to initialize your album, add the parameter "resize" and set it to "true".

portofolio("albumID", {
   type : "grid",
   resize : true
});

Swap (grid extension)

As an extension, swap is optional and is not natively supported in the Portofolio core files, which is why you'll need to download an extra file.

Swap is an extension for the grid album option that allows the user to swap grid items between them by drag and dropping them onto each other.
